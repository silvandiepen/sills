# Version-one acceptance criteria

- Detect at least 90% of seeded critical findings and 80% of seeded major findings in evaluated environments.
- Keep known-good false positives below 10%.
- Produce valid report JSON.
- Require evidence or explicit manual-review status for every issue.
- Never invent source locations.
- Never modify files outside the audit directory.
- List tested and untested areas.
- Produce comparable core findings in Claude Code and Codex.
- Produce a usable remediation handoff.
