# Security standards

Use the current applicable OWASP ASVS version for web application verification and the current OWASP MASVS categories for mobile applications. Record the exact version and requirement identifiers used in the report. Standards mapping supports traceability; it does not turn the audit into certification.

Also consider framework, platform, cloud, package-manager, and repository guidance when the project documents them. Project-specific security requirements take precedence when they are stricter.
