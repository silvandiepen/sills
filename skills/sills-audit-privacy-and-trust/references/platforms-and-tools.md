# Platforms and tools

Use source and configuration inspection, runtime network evidence, permission manifests, SDK and cookie inventories, policy documents, account controls, logs and schemas where authorised, and app-store privacy metadata. Keep personal data and secrets out of reports.

## Tool rules

- Record tool name, version, configuration, environment, and timestamp.
- Preserve raw output under the audit directory when safe.
- Prefer repeatable scripts and machine-readable output.
- Treat tool failures and partial results as limitations.
- Do not install into or mutate the audited project.

## Limitations

This is not legal advice, regulatory certification, or proof of privacy compliance. Data may flow through unavailable infrastructure or processors. Report jurisdiction, production, contractual, retention, backup, and human-process gaps explicitly.
