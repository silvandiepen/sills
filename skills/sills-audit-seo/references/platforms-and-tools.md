# Platforms and tools

Use HTTP responses, rendered browser evidence, sitemap and robots parsing, link graphs, structured-data validation, performance context, analytics or search-console evidence when supplied, and source inspection. Do not submit URLs, change search-console settings, or modify production.

## Tool rules

- Record tool name, version, configuration, environment, and timestamp.
- Preserve raw output under the audit directory when safe.
- Prefer repeatable scripts and machine-readable output.
- Treat tool failures and partial results as limitations.
- Do not install into or mutate the audited project.

## Limitations

An audit cannot guarantee indexing, ranking, traffic, or rich results. Search systems change and may ignore valid signals. State the crawl scope, user agent, locale, authentication, rendering, field-data, and external-tool limitations.
