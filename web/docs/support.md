# Support

Sills is an open source project. The best support path depends on what you need.

## Questions and issues

Open an issue in the [GitHub repository](https://github.com/silvandiepen/sills) for installation problems, documentation gaps, package behavior, or audit-output questions.

Do not include secrets, private source code, credentials, customer data, or unpublished audit evidence in public issues.

## Security-sensitive reports

For security-sensitive concerns, avoid posting exploit details, secrets, credentials, private logs, or private customer information publicly. Open a minimal GitHub issue asking for a private contact path, or use the repository security reporting flow if one is available.

## What to include

- The Sills package and version you installed.
- The command you ran.
- The agent surface you used: Codex, Claude Code, CI, or another environment.
- The audit mode and depth, if relevant.
- The exact error message or report section, with private information removed.

## Project scope

Sills can help produce audit reports, evidence records, and handoff notes. It does not provide legal advice, compliance certification, penetration testing, or a guarantee that a product is secure, accessible, compliant, complete, or defect-free.
