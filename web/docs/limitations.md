# Limitations

Sills cannot prove that a product is fully accessible, secure, usable, well-written, or well-architected. Automated tools cover only part of each discipline. Runtime access, roles, platforms, assistive technologies, source maps, documentation, and environment permissions affect coverage.

Security audits are passive reviews, not penetration tests. Accessibility mappings are technical, not legal advice. Design observations include informed judgement and are labelled accordingly.
