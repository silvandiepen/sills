# Audit trends

Trend reports compare findings by stable identity, not title text or array position. They report previous, current, resolved, new, unchanged, improved, and regressed findings.

A trend is only valid when the baseline scope is comparable. Scope changes, missing runtime roles, or reduced capabilities must be called out before totals are interpreted.

Trend data is written to `trend.json` and summarized in the human report without replacing the current audit conclusion.
